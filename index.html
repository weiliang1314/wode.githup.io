穷举问题——求出{a,b,c}的所有子集

什么是穷举

根据所列出的问题逐一列举问题所涉及的所有情形，并根据问题提出的条件排查哪些是问题的解，哪些应予排除。

问题分析

  • 根据数学推理可以得出有2的3次方个解集，但根据问题我们要把每种情况都列举出来，那么就需要对每一个元素是否取出进行决策，就需要使用递归去重复决策每种情况*

    代码实现

    let find = function(a, decisions) {
    if (a.length === decisions.length) {

     let dec = [];
 for (let i = 0; i < a.length; i++) {
     //根据决策是true或false拿出
     if (decisions[i]) {
         dec.push(a[i])
     }

   }
 return dec.join('')
}
//递归进行决策
 let r = [];
 r = r.concat(find(a, decisions.concat(true)))
 r = r.concat(find(a, decisions.concat(false)))
return r;
 }

    全排列问题——求出’abc’的全排列

    什么是全排列

  • 从n个不同元素中任取m（m≤n）个元素，按照一定的顺序排列起来，叫做从n个不同元素中取出m个元素的一个排列；当m=n时所有的排列情况叫全排列。*

    问题分析

  • 由数学推理可知共有3的阶乘排列可能，根据问题我们需要对abc都做出决策，且不能取之前已经取过的 *

    代码实现

    let permutation = function(s, decisions) {
    if (s.length === decisions.length) {

    return decisions

    }
    let r = []

    //判断是否包含指定的子字符串

    for (let i of s.split(‘’).filter(x => !(decisions.includes(x)))) {

    r = r.concat(permutation(s, decisions + i))

    }
    return r;
    }

    组合问题——求出从’abc’中取出k个元素的所有组合

    什么是组合

  • 从n个不同元素中，任取m(m≤n）个元素并成一组，叫做从n个不同元素中取出m个元素的一个组合；从n个不同元素中取出m(m≤n）个元素的所有组合的个数，叫做从n个不同元素中取出m个元素的组合数*

    问题分析

  • 由数学推理可知共有C(3,k),（k<=3）种情况；根据问题所取得元素如果为0或与所提供的元素个数相同那么就只有一种情况；我们可以先从提供的元素中取一个再从剩下取k-1个，第一个元素的情况取完后，从第二个元素开始又往后取…*

    代码实现

    let combination = function(str, k) {
    if (k === 0 || k === str.length) {

    return [str.slice(0, k)]

    }
    let r = []
    const [first, …other] = str;
    r = r.concat(combination(other, k - 1).map(x => [first, …x]))
    r = r.concat(combination(other, k))
    return r

    }

代理模式

定义：给某对象提供一个代理以控制对该对象的访问，访问对象不适合或者不能直接引用目标对象，代理对象作为访问对象和目标对象之间的中介。
特点：在客户端与目标对象之间起到一个中介作用和保护目标对象的作用；可以扩展目标对象的功能；将客户端与目标对象分离，降低系统的耦合度，增加了程序的可扩展；会造成请求处理速度变慢

问题：二维数组由0，1组成，把有0的整行，整列都变为0
问题分析：首先我们要考虑第一行，第一列是否有0，若有0则就要把其所在行或列都置为0；判断其余行列是否有0，若有则要把其所在行和列的第一个都置为0，再把第一行和第一列有0的都置为0

 let setZero = function(matrix) {
//检查第一行，第一列是否有0，

            matrix[row][0] = 0;

         }
       }

什么是设计模式

*设计模式是一套被反复使用的、经过分类编目的、解决代码设计问题的经验。使用设计模式是为了重用代码、让代码更容易被他人理解、保证代码可靠性。 项目中合理地运用设计模式可以完美地解决很多问题，每种模式在现实中都有相应的原理来与之对应，每种模式都描述了一个在我们编写代码时不断重复发生的问题，以及该问题的核心解决方案。 *

设计模式特点

  1. 提供固定的解决方法来解决在软件开发中出现的问题
  2. 可以适应自己需要的开箱即用的解决方案
  3. 有一组结构和词汇可以非常优雅地帮助表达相当大的解决方案

    设计模式分类

  4. 创建型模式:对象实例化的模式；工厂方法模式、抽象工厂模式、单例模式、建造者模式、原型模式
  5. 结构型模式:类或对象的组合；适配器模式、装饰器模式、代理模式、外观模式、桥接模式、组合模式、享元模式
  6. 行为型模式:描述类或对象怎样交互和分配职责；策略模式、模板方法模式、观察者模式、迭代子模式、责任链模式、命令模式

    设计模式六大原则

    单一职责原则

定义：一个类只负责一个功能领域中的相应职责，或者就一个类而言，应该只有一个引起它变化的原因

开闭原则

定义：一个软件实体应当对扩展开放，对修改关闭。即软件实体应尽量在不修改原有代码的情况下进行扩展

里氏代换原则

定义：所有引用基类（父类）的地方必须能透明地使用其子类的对象

依赖倒置原则

定义：高层模块不应该依赖低层模块，两者都应该依赖其抽象；抽象不应该依赖细节，细节应该依赖抽象，其核心思想是：要面向接口编程，不要面向实现编程

接口隔离原则

定义：使用多个专门的接口，而不使用单一的总接口，即客户端不应该依赖那些它不需要的接口

迪米特法则

定义：一个软件实体应当尽可能少地与其他实体发生相互作用

最长回文子字符串

问题：字符串中的子串，从前到后或从后到前都是一样的，例：bab；输出最长的回文子串
问题分析：首先我们要考虑字符串只有一个字符即length=1,这无法进行判断；当length>2,遍历字符串，字符串长度为偶数时
判断当前位置字符i及i+1是否相同,相同然后以这两个为中心向两边扩展判断(记录位置left,right)，记录最长回文子串开始位置left,长度length=right-left+1;字符串长度为奇数时，以当前位置i为中心向两边扩展判断

代码实现

let longlength = function(s) {
if (s.length < 2) {
   return s;
    };
    let start = 0; //记录最大子字符串起始位置
  let maxlength = 1; //若只有两个字符，则无法一中心向左右扩散；记录最大子串长度
  function expandAroundCneter(left, right) {
   while (left >= 0 && right < s.length && s[left] == s[right]) {
       //用下标相减得不到子串长度
       if (right - left + 1 > maxlength) {
           maxlength = right - left + 1;
           start = left;

       }
       left--;
       right++;
        }
        }
     for (let i = 0; i < s.length; i++) {
   expandAroundCneter(i - 1, i + 1);
   expandAroundCneter(i, i + 1) //如果字符串长度是偶数
      }
      return s.substring(start, start + maxlength) //截取字符串
    }

防抖函数

描述：当持续触发事件时，n秒内没有再触发事件，事件处理函数才会执行一次，如果设定的时间到来之前，又一次触发了事件，就重新开始延时。
用途：搜索联想，用户在不断输入值时，用防抖来节约请求资源。

代码实现

let input = document.querySelector('input');

function debounce(func, delay) {
let timer = null;

 return function(...args) {
    if (timer) clearTimeout(timer);

    timer = setTimeout(() => {
        //有参数时
        func.apply(this, args); //func();
           }, delay)
        }
   }

节流函数

描述：连续触发事件但是在n秒中只执行一次函数。
用途：鼠标连续不断地触发某事件，只在单位时间内只触发一次；在页面的无限加载场景下，需要用户在滚动页面时，每隔一段时间发一次请求，而不是在用户停下滚动页面操作时才去请求数据。

代码实现

function throttle(fn, delay) {
let timer;
return function(...args) {
    if (timer) {
        return;
       }
    timer = setTimeout(() => {
        // fn.apply(this, arguments);或
        //三点运算符，代替arguments:传递给函数的隐式参数，调用传递给函数fn实参
        fn(...args);

        timer = null;
           }, delay)
         }

     }

打家劫舍问题（动态规划）

问题：给定一个数组，代表每个房屋存放的金额，当偷相邻两个房屋时就会报警，计算在不触发报警系统的情况下偷盗的最大金额
问题分析：数组中的值为1个或0个则问题不成立，假设数组中有3个以上数值，所取得数不能相邻，当前索取金额总数=正在取+之前取,所取得金额总数最大;m[i]=n[i]+m[i-2]或m[i]=mi-1

代码实现

  let rob = function(nums) {
    //边界
    if (nums.length === 0) {
        return 0
    }
    if (nums.length === 1) {
        return nums[0]
    }
    const memo = [];
    memo[0] = nums[0];
    memo[1] = Math.max(nums[0], nums[1]);
    for (let i = 2; i <= nums.length; i++) {
        memo[i] = Math.max(nums[i] + memo[i - 2], memo[i - 1]);

    }
    return memo[nums.length - 1];
}

螺旋矩阵问题

问题：输入一个二维数组，按照从左向右，再向下，再向左，再向上，向右的方式遍历输出
问题分析：因为输入二维数组，按照顺时针遍历输出，那么首先就要要考虑边界问题；left最开始0；right为m0.length-1(m为输入的二维数组)，top为0，bottom为m.length-1;从左向右遍历不能超过右边界，从上到下不能超过下边界

代码实现

   let spiralOrder = function(matrix) {
  if (matrix.length == 0) {
    return [];
  }
  let left = 0;
  let right = matrix[0].length - 1;
 let top = 0;
  let bottom = matrix.length - 1;
  let dir = "right";
  let result = [];
  while (left <= right && top <= bottom) {
    if (dir == "right") {
        for (let i = left; i <= right; i++) {
            result.push(matrix[top][i])
        }
        top++;
        dir = "down"
    } else if (dir == "down") {
        for (let i = top; i <= bottom; i++) {
            result.push(matrix[i][right])
        }
        right--;
        dir = 'left';
    } else if (dir == "left") {
        for (let i = right; i >= left; i--) {
            result.push(matrix[bottom][i])
        }
        bottom--;
        dir = "top"
    } else if (dir == "top") {
        for (let i = bottom; i >= top; i--) {
            result.push(matrix[i][left])
        }
        left++;
        dir = "right"
    }
}
return result;

}

加油站问题（贪心）

贪心算法：从某一个初始状态出发，每次通过选取局部最优解向目标前进，并最终期望取得整体最优解的一种算法。由这个定义可知，贪心选择标准就是选择“当前最好”的决策，贪心算法根据这个标准进行决策，将原问题变成一个相似但规模更小的子问题，而后每一步选出来的一定是原问题整体最优解的一部分。如果一个问题贪心后只剩下一个子问题且有最优子结构，那么该问题就可以使用贪心算法。当一个问题的整体最优解包含其子问题的最优解时，我们称次问题具有最优子结构性质。
问题：环形路上有n个加油站，其中第i个加油站有get[i]升油， 有一辆无限容量油箱的车，从i加油站到i+1需要消耗cos[i]升油从其中一个加油站出发，开始时油箱没油，环形一周，返回出发时加油站编号
问题分析：首先要考虑所有加油站的油量是否多于或等于整个路程所消耗的油量；在此问题中假设汽车开始位置s,当前油量c，汽车开始时油量
为0；判断从i加油站到i+1加油站汽车油量是否足够，行驶后油量：c=c+get[i]-cos[i]

代码实现

let canComolete = function(gas, cost) {
let total = 0; //加油站总油
let totalcost = 0; //消耗总油
for (let i = 0; i < gas.length; i++) {
    total += gas[i];
    totalcost += cost[i]
}
if (total < totalcost) {
    return -1
}
let current = 0;
let start = 0; //假设的初始加油站
for (let i = 0; i < gas.length; i++) {
    //当前油量
    current = current - cost[i] + gas[i];
    if (current < 0) {
        current = 0;
        start = i + 1;
    }
}
return start; //出发时加油站编号
   }

最大正方形

问题：输入一个0和1组成的二维矩阵，找到只包含1的最大正方形，返回面积
问题分析：当1在边界时，最大正方形面积为1，边长d=1;当所选1的左边，上边，左上边，都不是0，它的
边长才能加1，当前边长=上一次最短边长+1

代码实现

let maxzheng = function(martix) {
let maxSidelength = 0; //记录最大边长

 let dp = new Array(martix.length);
for (let i = 0; i < martix.length; i++) {
    dp[i] = new Array(martix[i].length).fill(0); //把它变成一个二维数组，初始值为0
    for (let j = 0; j < martix[i].length; j++) {
        if (martix[i][j] == '1') {
            if (i == 0 || j == 0) { dp[i][j] = 1; } else {
                dp[i][j] = Math.min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1]) + 1 //左边，上边，左上
            }
            maxSidelength = Math.max(maxSidelength, dp[i][j])

        }
          }
            }
          return maxSidelength * maxSidelength;
          }

什么是动态规划算法

动态规划常常适用于有重叠子问题和最优子结构性质的问题，动态规划方法所耗时间往往远少于暴力解法。大致上，若要解一个给定问题，我们需要解其不同部分（即子问题），再合并子问题的解以得出原问题的解。通常许多子问题非常相似，为此动态规划法只解决每个子问题一次，从而减少计算量，一旦某个给定子问题的解已经算出，则将其存储，以便下次需要同一个子问题解之时直接查表。

三要素

  1. 最优子结构：子问题的最优解能够决定这个问题的最优解
  2. 边界：问题最小子集的解
  3. 状态转移函数：递推式

    解决爬楼梯问题

问题：一共有n阶楼梯，每次只能爬一阶或两阶，有多少种爬的方法
分析与解决：在这个问题中我首先考虑到当n=1时f(1)=1,当n=2时f(2)=2,但当n>2时发生了变化，f(3)=3,f(4)=5…
由此得出公式：f(n)=f(n-1)+f(n-2) n>2;f(1)与f(2)相当于边界，公式就是状态转移函数，f(n-1)与f(n-2)就是最优子结构

代码实现

function pa(n) {
   if(n<1){return 0;}
    if (n == 1 || n == 2) {
      return n;
      }
     let w = [];
        w[0] = 1;
       w[1] = 2;
        for (let i = 2; i < n; i++) {
       w[i] = w[i - 1] + w[i - 2];
      }
        return w[n - 1];
       }
