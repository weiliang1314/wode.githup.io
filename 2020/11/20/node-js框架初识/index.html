express介绍

Express 是一个简洁而灵活的 node.js Web应用框架, 提供了一系列强大特性帮助你创建各种 Web 应用，和丰富的 HTTP 工具。
使用 Express 可以快速地搭建一个完整功能的网站。

特性

  • 设置中间件来响应 HTTP 请求。
  • 定义路由表用于执行不同的 HTTP 请求动作。
  • 通过向模板传递参数来动态渲染 HTML 页面。

express使用

下载

npm install express

导入

() const express = require('express') const app = express(); ()

中间件简单使用

1、app.use放在首位,接收所有请求中间件,匹配所有路径，其中next()为继续向下执行。
(```)
app.use((req, res, next) => {
console.log(‘请求走了app.use中间件’)
next()

})

() *2、匹配指定的路径，当访问jun,走当前中间件。* ()
app.use(‘/jun’, (req, res, next) => {
console.log(‘请求走了app.use中间件，/jun’)
next()

})

() *3、定义访问错误信息。* ()
app.use((req, res, next) => {
//定义状态码
res.status(404)
res.send(‘当前页面内容不存在’)
})
() *4、登录路由保护，判断登录状态。* ()
app.use(‘/admin’, (req, res, next) => {
let lis = false;//true
if (lis) {
//请求继续向下
next()
} else {
res.send(‘没有登录不能访问’)

}

})
app.get(‘admin’, (req, res) => {
res.send(‘已经登录可以看了’)
})
(```)

捕获错误信息

1、
(```)
app.get(‘/index’, (req, res, next) => {
//抛出一个错误
throw new Error(‘未知错误’)
})
//错误处理信息
app.use((err, req, res, next) => {
//收到上面的错误信息
res.status(500).send(err.message)

})

() *2、捕获异步函数及同步代码在执行过程中的错误。* ()
app.get(‘/index’, async(req, res, next) => {
try { await readFile(‘错误的文件’) } catch (ex) {
//传递错误信息到终端
next(ex)
}

})

(```)

express模板引擎 express-art-template

第三方模块下载
npm install art-template express-art-template
导入
const expressArtTemplate = require('express-art-template')
使用express-art-template渲染后缀为art文件模板
app.engine('art', expressArtTemplate)
模板存放位置,第一个views告诉模板引擎模板位置，第二个是文件夹的位置
app.set('views', path.join(__dirname, 'views'))
告诉框架模板默认后缀
app.set('view engine', 'art')
模板拼接
() app.get('/index', (req, res) => { //要拼接的模板文件名，数据 res.render('index', { 数据 }) }) ()
